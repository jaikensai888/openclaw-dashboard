# Openclaw Dashboard - 产品需求文档

*版本：1.0 | 状态：已确认 | 项目类型：已有项目*

## 1. 产品概述

### 1.1 产品定位

一个为 Openclaw Agent 提供的 Web 聊天界面，类似于 ChatGPT，支持实时流式输出和任务追踪。

### 1.2 项目背景

| 项目信息     | 描述                                                                 |
| ------------ | -------------------------------------------------------------------- |
| 技术栈       | Next.js 14 + Fastify + SQLite + WebSocket                            |
| 现有功能概述 | 会话管理、实时聊天、流式响应、任务追踪、Dashboard 插件集成            |
| 本次目标     | 完善现有功能（Phase 4）：历史消息加载、错误处理、UI 优化、响应式布局  |

### 1.3 核心痛点

1. **缺乏统一的 Web 界面** - Openclaw Agent 主要通过 Telegram/Discord 等渠道交互，缺乏直接的 Web 聊天入口
2. **任务状态不透明** - Agent 执行耗时操作时，用户难以了解进度和状态
3. **历史记录管理困难** - 缺乏会话历史管理和消息持久化能力

### 1.4 核心价值

- **统一入口**：提供一个专门的 Web 界面直接与 Openclaw Agent 交互
- **实时反馈**：支持流式响应和任务进度追踪，让用户了解 Agent 正在做什么
- **历史管理**：持久化会话和消息，支持历史查看和回顾

### 1.5 MVP 目标

完成一个可用的 Web 聊天界面，支持：
- 创建、切换、删除会话
- 发送消息并实时接收流式响应
- 查看 Agent 执行任务的进度和详情
- 浏览历史会话和消息

| AI 爱好者    | 对 AI Agent 感兴趣的早期使用者             | 了解 Agent 的执行过程和任务状态    | 探索 Agent 能力、调试提示词          |

## 3. 功能范围

### 3.1 现有功能 🔄

> 以下功能已实现（Phase 1-3 完成）

- **会话管理** - 创建、删除、切换会话，会话列表展示
- **实时聊天** - 发送消息，WebSocket 实时通信
- **流式响应** - Agent 响应实时流式输出
- **任务追踪** - 任务卡片展示，任务状态更新
- **任务详情** - 点击卡片查看任务详情模态框
- **Dashboard 插件** - 与 Openclaw 的 Channel 插件集成
- **消息持久化** - SQLite 存储会话和消息
- **会话置顶** - 支持置顶重要会话

### 3.2 MVP 范围内 ✅

> Phase 4 需要完善的功能

- **历史消息加载** - 页面加载时获取历史消息，支持分页
- **断线重连** - WebSocket 断开自动重连，状态恢复
- **错误处理** - 友好的错误提示，消息发送失败重试
- **UI 优化** - 动画过渡、暗色模式完善、细节打磨
- **响应式布局** - 适配不同屏幕尺寸

### 3.3 MVP 范围外 ❌

- **多用户/认证系统** - 个人使用，无需多用户支持
- **多语言支持** - 中文优先，暂不支持国际化
- **移动端适配** - 桌面优先，移动端作为后续优化
- **媒体文件上传** - MVP 阶段暂不支持图片、文件上传
- **消息搜索** - 后续版本考虑
- **会话分享/导出** - 后续版本考虑
- **消息编辑/删除** - 后续版本考虑

## 5. 数据流转

```
用户输入消息
    ↓
前端 WebSocket 发送 chat.send
    ↓
Dashboard Backend 接收并存储消息
    ↓
转发到 Openclaw Dashboard Plugin
    ↓
Openclaw Agent 处理消息
    ↓
Agent 流式响应 → Plugin → Backend → 前端实时显示
    ↓
（如涉及任务）Agent 输出任务标记 → Backend 解析 → 创建/更新任务 → 前端展示任务卡片
    ↓
响应完成，消息持久化到 SQLite
```

## 6. 成功标准

### 6.1 完成标准（Definition of Done）

- [ ] 所有 P0 用户故事验收标准通过
- [ ] 所有 P1 用户故事验收标准通过
- [ ] 前后端核心功能测试通过
- [ ] WebSocket 连接稳定，支持断线重连
- [ ] 界面在主流浏览器（Chrome、Firefox、Safari）正常显示
- [ ] 消息发送、接收、持久化流程完整可用

### 6.2 成功指标（KPI）

- **响应速度**：用户发送消息后，100ms 内显示发送状态
- **流式延迟**：Agent 响应的第一个字符在 500ms 内显示
- **连接稳定性**：WebSocket 断线后 5s 内自动重连
- **数据完整性**：消息和会话数据不丢失

## 7. 风险与约束

| 风险/约束                       | 影响程度 | 应对策略                                     |
| ------------------------------- | -------- | -------------------------------------------- |
| Openclaw Agent 依赖             | 高       | Dashboard 可独立运行，Agent 故障不影响已存储数据 |
| 网络不稳定导致 WebSocket 断开   | 中       | 实现自动重连机制，断线期间缓存消息           |
| 任务协议变更                    | 中       | 协议解析器独立模块，易于更新                 |
| 大量历史消息加载性能            | 低       | 实现分页加载，限制单次加载数量               |
| SQLite 并发写入限制             | 低       | 个人使用场景，并发压力小                     |

## 8. 任务协议说明

Agent 通过内联标记报告任务状态，Dashboard 后端解析这些标记：

| 标记 | 格式 | 说明 |
|------|------|------|
| 开始 | `[TASK:START:type:title]` | 开始新任务，type 为任务类型 |
| 进度 | `[TASK:PROGRESS:percent:message]` | 更新进度（0-100）和消息 |
| 完成 | `[TASK:DONE]` | 任务成功完成 |
| 失败 | `[TASK:FAILED:error]` | 任务失败 |

任务类型：`research` | `code` | `file` | `command` | `custom`
